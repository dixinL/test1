# 申万二级行业日报 —— 业务逻辑与运行方法论

## 一、业务背景与目标

### 1.1 为什么要做这个日报
在 A 股市场中，行业轮动是获取超额收益的核心来源之一。但市场上有 100+ 个申万二级行业，人工跟踪每天的变化既耗时又容易遗漏。

这个每日量化日报的核心目的是：**用数据化的方式，每天早上 5 分钟快速掌握全市场行业的整体状态和结构性机会。**

### 1.2 业务目标
- **感知市场温度**：当天市场是强还是弱？赚钱效应如何？
- **识别行业强弱**：哪些行业在走强？哪些在走弱？
- **判断拥挤程度**：热门行业是不是太拥挤了？还能不能追？
- **发现轮动信号**：有没有新的行业正在从底部启动？

### 1.3 核心分析框架：三维度评估法
日报的业务逻辑建立在三个维度的交叉验证之上：

| 维度 | 业务含义 | 怎么用 |
|------|----------|--------|
| **价格趋势** | 行业指数涨了还是跌了 | 看方向：已经涨起来的才有趋势 |
| **市场宽度** | 行业内有多少股票在均线上方 | 看广度：是龙头涨还是普涨 |
| **行业拥挤度** | 换手率、成交额在历史上的位置 | 看风险：太拥挤容易回调，太冷清可能是底部 |

三个维度结合，才能给出相对可靠的判断：
- **价格涨 + 宽度高 + 拥挤度低** = 健康上涨，可以关注
- **价格涨 + 宽度低 + 拥挤度高** = 仅龙头行情，情绪过热，谨慎追高
- **价格跌 + 宽度低 + 拥挤度低** = 冰点期，可能在筑底
- **价格跌 + 宽度高 + 拥挤度高** = 普跌+放量，风险还没释放完

## 二、日报的内容结构

### 2.1 一份日报里有什么

每天的日报（`daily_YYYY-MM-DD.html`）包含以下几个板块，从上到下依次排列：

```
┌─────────────────────────────────┐
│  标题 + 数据日期                 │
├─────────────────────────────────┤
│  【市场概览】                    │
│  ├─ 当前市场判断（一句话结论）    │
│  ├─ 市场宽度均值                 │
│  ├─ 行业拥挤度均值               │
│  ├─ 信号统计                     │
│  └─ 有效行业数                   │
├─────────────────────────────────┤
│  【市场热力图】                   │
│  ├─ 均线市场宽度热力图           │
│  └─ 行业拥挤度热力图             │
├─────────────────────────────────┤
│  【详细分析报告】                 │
│  ├─ 市场宽度分析                 │
│  ├─ 拥挤度分析                   │
│  ├─ 场景判断（4种场景）          │
│  ├─ 趋势信号统计                 │
│  └─ 重点行业点评                 │
└─────────────────────────────────┘
```

### 2.2 每个板块的业务价值

**① 市场概览 —— 30 秒知道今天市场怎么样**
- 打开日报先看顶部的"市场判断"和几个核心数字
- 相当于"每日开盘前的市场体温测量"

**② 热力图 —— 一眼看出强弱分布**
- 宽度热力图：暖色（红/黄）= 强势，冷色（绿/蓝）= 弱势
- 拥挤度热力图：颜色越深 = 越拥挤
- 两个图对比看，可以快速找到"宽度高 + 拥挤度低"的黄金区域

**③ 详细报告 —— 深度分析和具体结论**
- 给那些想深入了解的人看的
- 包含场景判断、信号列表、行业点评等

## 三、业务运行节奏

### 3.1 一天的时间线

| 时间（北京时间） | 发生什么 | 业务含义 |
|-----------------|----------|----------|
| **凌晨 5:00** | 自动开始生成日报 | 利用盘前时间，确保开盘前能看到 |
| 凌晨 5:02 | 数据拉取完成 | 前一交易日的所有数据都到手了 |
| 凌晨 5:04 | 分析报告生成 | 文本分析出结论 |
| 凌晨 5:05 | 热力图生成 | 可视化完成 |
| 凌晨 5:06 | HTML 整合完成 | 日报网页版就绪 |
| 凌晨 5:07 | RSS 更新 + 微信推送 | 订阅用户收到通知 |
| 凌晨 5:08 | 提交到 GitHub，Pages 部署 | 网页可以访问了 |
| **早上开盘前** | 用户收到推送，打开日报 | 5 分钟快速了解市场状态 |

> 整个过程大约 5-8 分钟，用户早上起来刷牙洗脸的功夫，日报已经准备好了。

### 3.2 数据的时间口径
- **数据截止**：前一个交易日收盘后的数据
- **生成时间**：当天凌晨 5 点
- **用途**：当天开盘前的参考，以及前一天的复盘

为什么是凌晨 5 点而不是晚上？
- 晚上交易所的数据同步可能有延迟
- 凌晨生成，早上起来看是最新的
- 不影响晚上做其他事情

## 四、核心业务逻辑

### 4.1 市场宽度怎么指导决策

**业务含义**：市场宽度 = 行业内有多少比例的股票站在均线上方。

**怎么读：**
- **宽度 > 80%**：行业内绝大多数股票都在均线上，普涨行情，趋势健康
- **宽度 50%-80%**：分化行情，部分股票在涨
- **宽度 < 30%**：行业内大部分股票都在均线下方，弱势区间

**进阶用法：价格新高但宽度没新高 = 背离，小心回调**

### 4.2 拥挤度怎么指导决策

**业务含义**：拥挤度 = 当前换手率/成交额在历史上的分位数位置。

**怎么读：**
- **拥挤度 > 80 分位**：太热了，大家都在买，短期可能回调
- **拥挤度 40-60 分位**：温度适中，健康状态
- **拥挤度 < 20 分位**：太冷了，没人关注，可能是底部区域

**进阶用法：宽度上升 + 拥挤度低 = 刚启动，值得关注**

### 4.3 四种市场场景

日报会根据宽度和拥挤度的组合，将市场分为四种场景：

| 场景 | 宽度 | 拥挤度 | 业务含义 | 操作建议 |
|------|------|--------|----------|----------|
| **场景 1：健康上涨** | 高 | 中低 | 普涨但还没到情绪极致 | 可以持有，关注领涨行业 |
| **场景 2：过热警惕** | 高 | 高 | 大家都冲进去了，容易回调 | 减仓追高的，别太乐观 |
| **场景 3：底部区域** | 低 | 低 | 没人关注，可能在磨底 | 可以左侧布局，但要等信号确认 |
| **场景 4：风险释放** | 低 | 高 | 下跌还放量，恐慌中 | 别急着抄底，等风险释放完 |

### 4.4 趋势信号是什么

日报会统计每个行业的"趋势信号"，本质上是一些技术形态的筛选：
- 均线多头排列
- 突破平台
- 底背离
- 等等

**业务价值**：从 100+ 个行业中自动筛选出技术形态值得关注的，节省人工翻图的时间。

## 五、产品形态与分发

### 5.1 三种使用方式

| 方式 | 适合人群 | 特点 |
|------|----------|------|
| **网页版** | 大多数用户 | 图文并茂，手机电脑都能看，地址不变 |
| **RSS 订阅** | 重度资讯用户 | 在 RSS 阅读器中收到更新提醒，不遗漏 |
| **微信推送** | 想要及时通知的用户 | 每天早上一条消息，带摘要，点进去看详情 |

### 5.2 RSS 设计的业务考量

**为什么要有 RSS？**
- 很多投资者有每天读资讯的习惯
- RSS 阅读器可以聚合多个信息源，一站式阅读
- 比打开网页更轻量

**pubDate 为什么只更新最新一条？**
- RSS 阅读器靠 pubDate 判断是不是新内容
- 如果每次重新生成都更新所有记录的时间，阅读器会认为所有日报都是新的，刷屏
- 只更新最新一条的时间：新内容会被识别，历史记录保持原样，用户体验好

### 5.3 微信推送的业务考量
- 微信是最高频的 App，触达率最高
- 推送内容是摘要，不是全文，避免信息过载
- 用户感兴趣再点链接看完整报告

## 六、可靠性与业务连续性

### 6.1 数据获取不到怎么办
系统设计了多层 fallback 机制，确保在数据 API 不稳定时也能产出可用的日报：

```
第一层：直接用本地已有的今天数据（最快）
   ↓ 没有
第二层：调用 API 实时拉取
   ↓ API 挂了
第三层：用昨天的数据顶上（至少有内容）
```

**业务逻辑**：有最新数据看最新的，没有就看最近可用的，总比什么都没有强。

### 6.2 某个模块挂了怎么办
采用"核心内容必须有，增强内容可降级"的策略：

| 模块 | 失败了怎么办 | 对用户的影响 |
|------|-------------|-------------|
| 分析报告 | 整体标记失败 | 影响大，核心内容没有了 |
| 宽度热力图 | HTML 里不显示这张图 | 影响小，还能看文字和拥挤度图 |
| 拥挤度热力图 | HTML 里不显示这张图 | 影响小，还能看文字和宽度图 |
| 排名功能 | HTML 里不显示排名 | 已移除，不影响核心功能 |

**业务逻辑**：热力图没了还有文字报告，排名没了还有分析，核心的市场判断不能丢。

### 6.3 为什么不用实时数据
- 日报是"盘前参考"和"昨日复盘"工具，不是实时盯盘工具
- 前一天收盘后的数据已经足够做行业级别的判断
- 用离线数据生成，速度快、成本低、稳定性高

## 七、业务方法论总结

这个日报系统的设计，背后遵循几条业务层面的方法论：

### 1. 信息分层，逐层深入
- 最上面是一句话结论 → 30 秒看完
- 然后是关键指标和热力图 → 3 分钟看完
- 最后是详细报告 → 想深挖的人慢慢看

用户可以根据自己的时间和需求，选择看到哪一层。

### 2. 数据说话，不做主观预测
- 所有结论都基于客观数据（宽度、拥挤度、价格）
- 给出的是"当前状态是什么"，而不是"明天会怎么样"
- 用户根据状态自己做决策，系统只提供信息

### 3. 自动化 + 定时送达，减少决策摩擦
- 每天自动生成，不用手动跑
- 自动推送到微信/RSS，不用主动去查
- 降低"每天看市场"的门槛，培养复盘习惯

### 4. 历史可追溯，方便复盘
- 每天的日报都保存下来，按日期命名
- 可以回头看当时的判断对不对，市场是怎么演化的
- 相当于一个"行业市场日记"

### 5. 宽度 + 拥挤度 双轮驱动
- 只看价格容易追高杀跌
- 加上宽度（广度）和拥挤度（情绪），判断更立体
- 本质上是在回答三个问题：涨了吗？涨得健康吗？还能涨多久？

## 八、适用场景与局限性

### 适用场景
- ✅ 中线行业轮动的参考
- ✅ 每天盘前快速了解市场状态
- ✅ 发现被忽略的底部行业
- ✅ 避免在过热的时候追高

### 局限性
- ❌ 不做个股推荐，只到行业级别
- ❌ 不预测涨跌，只描述当前状态
- ❌ 不是短线交易信号，偏中线视角
- ❌ 历史规律不代表未来，需要结合宏观环境判断

---

## 九、一句话总结

> 这不是一个"告诉你买什么"的系统，而是一个"每天帮你把 100 多个行业的状态整理好，让你 5 分钟就能对全市场有个整体感觉"的系统。它省的是时间，决策还是要你自己来做。
